# Codex Thread Continuity

Purpose: preserve day-to-day continuity for active Codex work so implementation, review state, and pending tasks can be resumed without reconstructing context from memory.

Daily rule:

1. At the end of each working day, write a short continuation note.
2. Record the active repository, branch, and current task.
3. Record the next concrete action for the next Codex session.
4. Record any blockers, missing credentials, or required user actions.
5. Reference the main files touched that day.

Minimum continuation entry template:

```text
Date:
Active repo:
Current task:
Status:
Next step:
Blockers:
Files touched:
```
