# Post-Freeze Policy

## Default Rule

After final freeze, this repository should be treated as an archive-first research record. Changes should be rare, explicit, and provenance-preserving.

## Allowed Post-Freeze Changes

- typo fixes that do not change authority meaning,
- manifest corrections with documented reason,
- additional provenance notes,
- human review notes,
- external archive metadata,
- security or privacy redactions with replacement notices.

## Restricted Post-Freeze Changes

These require a new audit note before modification:

- importing new model responses,
- moving files between authority domains,
- changing freeze scope,
- changing LSC/AOIA boundary language,
- modifying root orientation files,
- adding generated archive outputs.

## Forbidden Post-Freeze Changes

- silently rewriting model reports,
- deleting contradiction or contamination evidence,
- promoting model consensus to validation,
- reactivating old mixed roots as canonical sources,
- merging AOIA runtime source or LSC theory into MHLM/MDLH authority.
